Llama Bridge

A tiny tool that lets Claude hand off simple jobs to free AI models running on your own computer — and shows you the receipts.

01What is Llama Bridge?

Imagine Claude as a brilliant expert you pay by the word. Every word Claude writes costs money. But a lot of everyday work doesn't need an expert. Summarizing notes, sorting lists, cleaning up text — a smaller, free helper can do those jobs just fine.

That's where Ollama comes in. Ollama is a free app that runs smaller AI models right on your computer. They cost nothing to use. Llama Bridge is the connector between the two: when Claude hits a simple chore, it passes the chore to a local model, checks the answer, and keeps the savings on record.

It's like doing small household chores yourself instead of paying a contractor for every one of them. The contractor (Claude) still handles the wiring and the plumbing. The easy stuff stays in-house, for free — and Llama Bridge keeps track of every dollar you didn't spend.

One Python file. No installs beyond Python and Ollama. It runs an MCP server (so Claude can use it as a tool) and a web dashboard (so you can watch it work at http://127.0.0.1:8765).

02Why bother?

Four reasons people run it.

💰

Save money

Cloud AI charges for every token it writes. Local models write for free. Routine work adds up fast — especially repeated jobs like classifying hundreds of rows.

🏠

Keep data at home

Work sent to a local model never leaves your computer. For some jobs, that alone is the reason.

✅

Trust, but verify

Every handoff can be checked automatically: is it valid JSON? Does it include the right words? Is it long enough? Failed checks get retried on a bigger model.

📊

See the receipts

Every delegation is logged. The dashboard shows tokens saved, cost avoided, speed, and which models actually do good work.

03How it works

The whole trip takes a few seconds.

Claudespots a simple subtask
→
Llama Bridgepicks a model, sends the job
→
Ollamaruns it locally, free
→
Checksverify the answer
→
Claudeuses the result

Claude decides what to hand off

Only simple, well-defined chores get delegated: summarize, classify, extract, reformat, draft. The hard thinking stays with Claude.

The bridge picks the right model

It matches the job's family first (code jobs go to coder models, image jobs to vision models, everything else to general models), then picks the smallest size that can handle it.

The answer gets checked

The bridge runs automatic checks on the output. If a check fails, it retries once on a bigger model. If it still fails, the answer is flagged [UNVERIFIED] and Claude does the job itself instead.

Everything is logged

Each handoff is saved: the prompt, the answer, how long it took, and roughly what it saved. Repeat questions come back instantly from a cache. A scoreboard learns which models are good at which jobs.

04How to install it

Three steps. You need Python 3.8+ and about ten minutes.

Install Ollama and pull a starter model

Download Ollama from ollama.com and install it. Then open a terminal and pull a small model:

ollama pull llama3.2:3b

Get the MCP config from the bridge

Put llama_bridge.py in a folder it can stay in, then run:

python llama_bridge.py install

It prints a small block of JSON. Paste that into your Claude MCP config file (for the desktop app that's claude_desktop_config.json), or run the claude mcp add command it prints if you use Claude Code.

Restart Claude and start the dashboard

Restart Claude so it loads the new tools. Then start the dashboard to watch delegations happen live:

python llama_bridge.py

It opens at http://127.0.0.1:8765. Ask Claude to "check ollama_status" to confirm the connection works.

If Ollama is off or no model is installed, nothing breaks. Delegation is a bonus, not a requirement — Claude just does the work itself.

05Which models should you install?

The golden rule: match the model's family to the job first, then pick the smallest size that gets it done. Bigger is not always better — a mid-size general model usually beats a huge coding model at everyday writing tasks.

ModelSizeFamilyBest at
llama3.2:3b~2 GBgeneralQuick, high-volume chores: yes/no sorting, tiny reformats, simple labels. Start here.
qwen2.5:7b~4.4 GBgeneralEveryday summaries, drafts, extraction, and simple structured output.
gemma (12B class)~7 GBgeneralBroader knowledge, multi-step instructions, cleaner JSON. The reliable middle.
qwen3-coder:30b~17 GBcoderReal code and the hardest jobs. Slow — the bridge runs it in the background.
qwen3-vl / llama3.2-vision~6–7 GBvisionAnything with images or screenshots. Text-only models literally cannot see.

A good starter set is one small general model, one mid-size general model, and whichever specialist (coder or vision) matches your work. Your computer needs enough free RAM to hold the model — roughly the file size plus a little headroom.

08Common questions

Is it really free to run?

The local models are free to download and run. The "cost" is your computer's RAM and a few seconds per job. You still pay normally for what Claude itself does.

Does my data leave my computer?

Work delegated to a local model runs entirely on your machine. The conversation with Claude itself still works like it always does — the bridge only changes where the small jobs run.

What happens if Ollama is off?

Nothing bad. The bridge reports it can't reach Ollama, and Claude simply does the work itself. Delegation is an optimization, never a dependency.

Can I trust the local model's answers?

Trust the checks, not the model. Ask for verification (valid JSON, required words, minimum length) on anything that matters. Failed checks retry on a bigger model, and anything still failing is flagged [UNVERIFIED] so Claude redoes it.

Why did a job come back tagged [UNVERIFIED]?

The answer failed its checks even after a retry. Don't use it — that tag is the safety net catching a bad answer before it reaches you.

How big a computer do I need?

A 3B model runs on almost anything with 8 GB of RAM. Mid-size models (7–12B) like 16 GB. The 30B specialists want 24+ GB or a good GPU. Start small — small models handle most delegated work.

Where do I see what it's been doing?

Run python llama_bridge.py and open the dashboard. Every delegation is there: prompt, answer, time, tokens saved, and a 👍/👎 rating you can give so model picking keeps improving.
